# 前端接口文档

更新时间：2026-04-07

本文档基于当前 `frontend/` 目录中的真实代码整理，覆盖前端已经封装并实际接入的接口、统一请求规则、页面使用入口，以及本轮新增的校区绑定、草稿发布、校区偏好筛选相关能力。

## 1. 基础说明

### 1.1 Base URL

统一接口前缀定义在 `frontend/utils/request.js`：

```js
export const BASE_URL = 'http://localhost:8080/api/v1'
```

### 1.2 统一请求行为

- JSON 接口通过 `request(options)` 发起。
- 文件上传通过 `uploadFile(options)` 发起。
- 已登录时会自动注入请求头：

```http
Authorization: Bearer <token>
```

- 后端返回 HTTP `401` 时，前端会自动：
  - 清空本地登录态
  - Toast 提示“登录已过期，请重新登录”
  - `reLaunch` 到 `/pages/user/login`
- 网络失败时统一 Toast 提示。
- 页面层通常通过 `res.code === 0` 判断是否成功。

### 1.3 统一响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

说明：

- `code = 0`：成功
- `code != 0`：业务失败

### 1.4 分页结构

列表接口统一返回：

```json
{
  "records": [],
  "total": 0,
  "pageNum": 1,
  "pageSize": 10
}
```

### 1.5 前端本地状态

`frontend/store/auth.js`

- `token`
- `user_profile`

`user_profile` 当前常用字段：

- `id`
- `username`
- `campusCode`
- `campusName`

`frontend/store/goods.js`

- `keyword`
- `categoryId`
- `minPrice`
- `maxPrice`
- `quickFilter`
- `feedTab`
- `lastViewedId`
- `preferredCampusCode`

### 1.6 支持的校区选项

前端固定校区选项定义在 `frontend/utils/campus.js`：

| code | name |
|---|---|
| `NEUSOFT` | 广东东软学院 |
| `SCNU_NANHAI` | 华南师范大学佛山校区南海校区 |
| `GD_LIGHT_INDUSTRY_NANHAI` | 广东轻工职业技术大学南海校区 |

补充说明：

- 已登录用户默认优先使用账号绑定校区。
- 未登录用户可在首页和商品列表页通过 `preferredCampusCode` 手动设置浏览偏好校区。
- H5 定位默认使用 `wgs84`。
- 非 H5 端优先使用 `gcj02`，失败后回退到 `wgs84`。

## 2. 认证模块

代码文件：`frontend/api/auth.js`

### 2.1 登录

```js
login(data)
```

- 方法：`POST`
- 地址：`/auth/login`
- 是否鉴权：否
- 页面入口：`/pages/user/login`

请求示例：

```json
{
  "username": "demo_buyer",
  "password": "password"
}
```

成功返回 `data`：

```json
{
  "token": "jwt-token",
  "userId": 1,
  "username": "demo_buyer",
  "campusCode": "NEUSOFT",
  "campusName": "广东东软学院"
}
```

说明：

- 登录页会把 `token` 写入本地。
- 登录页还会把 `userId`、`username`、`campusCode`、`campusName` 一并写入 `user_profile`。

### 2.2 注册

```js
register(data)
```

- 方法：`POST`
- 地址：`/auth/register`
- 是否鉴权：否
- 页面入口：`/pages/user/register`

当前注册页实际提交：

```json
{
  "username": "new_user",
  "password": "123456"
}
```

说明：

- 后端额外支持 `phone`、`studentNo` 字段。
- 但当前前端注册页还没有这两个输入项，因此不会提交。

### 2.3 获取当前用户资料

```js
getProfile()
```

- 方法：`GET`
- 地址：`/auth/profile`
- 是否鉴权：是
- 页面入口：`/pages/user/profile`

成功返回 `data` 常用字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number | 用户 ID |
| `username` | string | 用户名 |
| `phone` | string | 手机号 |
| `studentNo` | string | 学号 |
| `campusCode` | string | 当前绑定校区编码 |
| `campusName` | string | 当前绑定校区名称 |
| `campusBindSource` | string | `MANUAL` / `LOCATION` |
| `campusBoundAt` | string | 校区绑定时间 |
| `role` | string | 当前固定为 `USER` |
| `onSaleCount` | number | 在售商品数 |
| `soldCount` | number | 已售商品数 |
| `orderCount` | number | 作为买家的订单数 |
| `favoriteCount` | number | 收藏总数 |
| `unreadMessageCount` | number | 站内消息未读数 |

### 2.4 手动绑定校区

```js
bindCampusManual(data)
```

- 方法：`PUT`
- 地址：`/auth/campus/manual`
- 是否鉴权：是
- 页面入口：`components/CampusBindingWidget.vue`

请求示例：

```json
{
  "campusCode": "NEUSOFT"
}
```

说明：

- 返回值就是最新的用户资料对象，结构同 `getProfile()`。

### 2.5 通过定位绑定校区

```js
bindCampusByLocation(data)
```

- 方法：`PUT`
- 地址：`/auth/campus/location`
- 是否鉴权：是
- 页面入口：`components/CampusBindingWidget.vue`

请求示例：

```json
{
  "latitude": 23.0123,
  "longitude": 113.1234,
  "coordType": "wgs84"
}
```

说明：

- 成功后返回最新用户资料对象。
- 个人页会同步更新本地 `user_profile` 和 `preferredCampusCode`。

### 2.6 通过定位解析校区

```js
resolveCampusByLocation(data)
```

- 方法：`POST`
- 地址：`/auth/campus/resolve`
- 是否鉴权：否
- 页面入口：首页、商品列表页的校区偏好选择组件

成功返回 `data`：

```json
{
  "campusCode": "NEUSOFT",
  "campusName": "广东东软学院"
}
```

说明：

- 该接口只负责“识别当前定位属于哪个校区”。
- 不会修改用户账号资料。

## 3. 商品与 AI 模块

代码文件：`frontend/api/goods.js`

### 3.1 获取商品列表

```js
getGoodsList(params)
```

- 方法：`GET`
- 地址：`/goods`
- 是否鉴权：否
- 页面入口：`/pages/index/index`、`/pages/goods/list`

请求参数：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `pageNum` | number | 否 | 页码，默认 `1` |
| `pageSize` | number | 否 | 每页条数，默认 `10`，前端常传 `20` 或 `40` |
| `keyword` | string | 否 | 标题/描述关键字搜索 |
| `sortMode` | string | 否 | `latest` / `price-low` / `price-high` / `hot` |
| `preferredCampusCode` | string | 否 | 校区偏好，用于同校区优先排序 |

返回 `records[]` 常用字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number | 商品 ID |
| `sellerId` | number | 卖家用户 ID |
| `categoryId` | number | 分类 ID |
| `campusCode` | string | 商品发布时冻结的校区编码 |
| `campusName` | string | 商品发布时冻结的校区名称 |
| `title` | string | 标题 |
| `description` | string | 描述 |
| `price` | number | 售价 |
| `originalPrice` | number | 原价 |
| `conditionLevel` | number | 成色，`1-10` |
| `status` | string | 商品状态 |
| `viewCount` | number | 浏览量 |
| `sellerName` | string | 卖家名称 |
| `categoryLabel` | string | 分类名称 |
| `imageUrl` | string | 主图 URL |
| `thumbUrl` | string | 缩略图 URL |
| `favoriteCount` | number | 收藏人数 |
| `isFavorite` | boolean | 当前用户是否已收藏 |
| `createdAt` | string | 创建时间 |
| `updatedAt` | string | 更新时间 |

说明：

- 后端公开列表只返回 `ON_SALE` 商品。
- 首页和列表页会结合 `preferredCampusCode` 做“同校区优先”展示。
- 当前前端不会在列表页直接使用卖家操作能力字段。


详情对象在商品列表字段基础上还会额外返回：

| 字段 | 类型 | 说明 |
|---|---|---|
| `images` | string[] | 完整图片列表 |
| `canEdit` | boolean | 卖家视角下是否允许编辑 |
| `canDelete` | boolean | 卖家视角下是否允许删除 |
| `canOnSale` | boolean | 卖家视角下是否允许重新上架 |
| `hasPendingPaymentOrder` | boolean | 是否存在待付款订单 |
| `editBlockedReason` | string | 编辑受阻原因 |
| `deleteBlockedReason` | string | 删除受阻原因 |
| `onSaleBlockedReason` | string | 上架受阻原因 |

说明：

- 非卖家查看时后端会自动累加浏览量。
- `DRAFT` 只允许卖家本人查看。
- 发布页编辑模式会根据 `canEdit` 和 `editBlockedReason` 决定是否允许进入编辑。

### 3.5 发布商品

```js
createGoods(data)
```

- 方法：`POST`
- 地址：`/goods`
- 是否鉴权：是
- 页面入口：`/pages/goods/publish`

请求示例：

```json
{
  "title": "九成新机械键盘",
  "description": "自用闲置，支持当面验货",
  "categoryId": 1,
  "price": 168,
```

### 3.13 上传商品图片

```js
uploadGoodsImage(filePath)
```

- 方法：`POST`
- 地址：`/files/images`
- 是否鉴权：是
- 请求类型：`multipart/form-data`
- 表单字段：`file`
- 页面入口：`/pages/goods/publish`

成功返回 `data`：

```json
{
  "url": "http://localhost:8080/uploads/goods/20260403/abc123.webp",
  "thumbUrl": "http://localhost:8080/uploads/goods/20260403/abc123_thumb.webp",
  "originalName": "keyboard.jpg",

```

说明：

- 单次上传 1 张图。
- 列表卡片优先使用 `thumbUrl`，商品详情和 AI 识别继续使用 `url`。
- 发布页在部分端上会先尝试压缩超大图片，再上传。

### 3.14 上传 AI 估价图片

```js
uploadAiValuationImage(filePath)
```

- 方法：`POST`
- 地址：`/ai/valuations/uploads`
- 是否鉴权：是
- 页面入口：`/pages/goods/publish`、AI 估价面板

成功返回 `data`：
```json
{
  "uploadToken": "7d9a3f8d3d4b4a42b2e6b8c6e3f0a123",
  "expiresAt": 1775555555000,
  "fileName": "keyboard.jpg",
  "contentType": "image/jpeg",
  "size": 245781
}
```

说明：
- 该接口只给 AI 估价做临时图片上传，不会把图片落到商品正式图存储中。
- 单次上传 1 张图，前端会按当前图片顺序逐张换取 `uploadToken`。
- `uploadToken` 默认 10 分钟过期，过期后前端会重新上传当前本地图再发起估价。

### 3.15 创建 AI 估价

```js
createAiValuation(data)
```

- 方法：`POST`
- 地址：`/ai/valuations`
- 是否鉴权：是
- 页面入口：`/pages/goods/publish`

请求示例：

```json
{
  "uploadTokens": [
    "7d9a3f8d3d4b4a42b2e6b8c6e3f0a123"
  ],
  "existingImageUrls": [
    "http://localhost:8080/uploads/goods/20260403/demo-2.webp"
  ]
}
```

说明：
- `uploadTokens` 与 `existingImageUrls` 合计最多 3 张图。
- 本地图会先换取 `uploadToken` 再创建 AI 估价；已上传的 HTTP 图片直接放到 `existingImageUrls`。
- 智能估价不再复用商品正式图片上传接口，保存草稿/正式发布时仍然走 `uploadGoodsImage`。
- AI 提示词只使用原图和服务端系统提示词，不会读取标题、描述、分类、价格等发布表单内容。

成功返回 `data` 主要字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `valuationId` | number | 估价记录 ID |
| `itemName` | string | 识别出的商品名称 |
| `brand` | string | 品牌 |
| `model` | string | 型号 |
| `categoryIdSuggest` | number | 建议分类 ID |
| `categoryNameSuggest` | string | 建议分类名称 |
| `conditionScore` | number | 成色评分 |
| `conditionText` | string | 成色文案 |
| `visibleIssues` | string[] | 可见问题 |
| `priceLow` | number | 价格下限 |
| `priceHigh` | number | 价格上限 |
| `suggestedPrice` | number | 建议售价 |
| `confidenceLevel` | string | `high` / `medium` / `low` |
| `suggestions` | string[] | 补充建议 |
| `fillSuggestion` | object | 表单回填建议 |
| `adopted` | boolean | 是否已采纳 |

## 9. 状态与枚举

### 9.1 商品状态

| 值 | 说明 |
|---|---|
| `DRAFT` | 草稿 |
| `PENDING` | 待审核 |
| `ON_SALE` | 在售 |
| `OFFLINE` | 已下架 |
| `SOLD` | 已售出 |
| `DELETED` | 已删除，仅后端内部状态 |

说明：

- 当前前端已经接入 `DRAFT`、`ON_SALE`、`OFFLINE`、`SOLD`。
- `PENDING` 状态位仍保留在页面筛选和数据结构中，但当前项目尚无真实审核流页面。

### 9.2 商品排序模式

| 值 | 说明 |
|---|---|
| `latest` | 最新发布 |
| `price-low` | 价格从低到高 |

| `hot` | 热度优先 |
